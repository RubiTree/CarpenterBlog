最后是列表，有个很简单的特性，但是对我的帮助会很大

那就是：列表项目可以包含多个段落，每个项目下的段落都必须有同样的缩进

列表的结束判断比你想象中复杂，于是你可以在列表中加入更多东西

你可能会说列表还不简单吗，我闭着眼睛都能写出来，那你可以先看这几个例子，看看它的渲染想过是跟你想的一样吗

【1】首先这是四行普通的列表

  1. 黄梅时节家家雨
  2. 青草池塘处处蛙
  3. 有约不来过夜半
  4. 闲敲棋子落灯花

【2】直接在列表间加入空行，多个空行

  • 现象：都不影响列表的连续，只是整个列表的间距会增加
  • 分析：
    • 连续空行会当成一个空行处理
    • 空行不会中断列表
    • 列表中的空行会让渲染时，每个列表项作为一个段落，用 <p> 标签包起来

  1. 黄梅时节家家雨

  2. 青草池塘处处蛙

  3. 有约不来过夜半

  4. 闲敲棋子落灯花

【3】在列表间加入一行没有序号的行

  • 现象：这行会被自动划归到上一行，中间会有一个空格
  • 分析：无序号行不会中断列表，或者说回车会被直接作为空格处理
  1. 黄梅时节 家家雨
  2. 青草池塘处处蛙
  3. 有约不来过夜半
  4. 闲敲棋子落灯花

【4】在[3]的基础上，在这行下侧加入一个空行

  • 现象：看分析
  • 分析：
    • 同样列表中的空行不会中断列表
    • 列表中的空行让每个列表项作为一个段落
    • 此时列表中的换行被处理成了段落内的换行
    • 于是在每个列表项作为一个段落之后，后面的列表项的换行会自动变成段落内换行，而不是变成空格
    • 多个换行同理

  1. 黄梅时节
    家家雨

  2. 青草池塘
    处处蛙

  3. 有约不来

【5】在[4]最简单情况的基础上，在这行的上方加入一个空行

  • 现象：列表终于被打断了，出现了两个列表
  • 分析：
    • 列表是被中间的段落打断的
    • 前后的空行符合一个段落的定义
  1. 黄梅时节

【6】在[5]的基础上，在文字前加1个空格

  • 现象：恢复成了一个列表，并且列表项内带了一个空行
  • 分析：
    • 中间的段落因为新加入的空格而被破坏，整个被作为列表项的一个段落处理，此列表项的段落中出现了空行

  1. 黄梅时节

【7】在[4]最简单情况的基础上，把下方的空行放在上方

  • 现象：列表被打断而且下方的列表渲染失败
  • 分析：
    • 首先下方的列表开头并没有空行，它不是个列表
    • 然后自然上方只剩下一个列表了，这样列表就被打断了

然后是列表和图片、代码段

一个 Markdown 段落是由一个或多个连续的文本行组成，它的前后要有一个以上的空行
（空行的定义是显示上看起来像是空的，便会被视为空行。比方说，若某一行只包含空格和制表符，则该行也会被视为空行）

「由一个或多个连续的文本行组成」这句话其实暗示了 Markdown 允许段落内强迫换行

文本处理逻辑只是markdown语法逻辑的实现，而这个markdown语法逻辑是基于一些符合人直觉的规则组成的
